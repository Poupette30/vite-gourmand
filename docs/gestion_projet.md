# Gestion de projet

## Objectif du projet

L'objectif est de développer une application web de gestion de commandes de menus pour un service traiteur.

Le projet répond au cahier des charges de l'ECF DWWM et comprend une base relationnelle, une base NoSQL, plusieurs rôles utilisateurs ainsi qu'un espace d'administration.

## Méthode
Le projet a été conduit selon une méthode agile simplifiée, organisée à l'aide d'un tableau Kanban comportant les colonnes :

- À faire
- En cours
- En test
- Terminé

Cette organisation a permis de suivre l'avancement des fonctionnalités tout au long du développement.

## Gestion du code source

Le projet est versionné avec Git et hébergé sur GitHub.

Le développement a été réalisé principalement sur la branche `main`. Les différentes fonctionnalités ont été ajoutées et testées progressivement au cours du développement avant d'être validées.

## User stories principales
- En tant que visiteur, je veux consulter les menus pour choisir une prestation.
- En tant qu’utilisateur, je veux commander un menu et suivre son statut.
- En tant qu’employé, je veux gérer les commandes afin d’organiser la production.
- En tant qu’administrateur, je veux créer et désactiver des employés.
- En tant qu’administrateur, je veux comparer les commandes par menu via un graphique.

## Critères d’acceptation

Chaque fonctionnalité est validée si le parcours fonctionne, les droits sont respectés, les erreurs sont gérées, et les données sont persistées.


## Organisation

Le projet a été découpé en plusieurs grandes étapes :

- analyse du besoin ;
- conception de la base de données ;
- développement de l'authentification ;
- développement des menus ;
- gestion des commandes ;
- espace employé ;
- espace administrateur ;
- statistiques MongoDB ;
- tests ;
- documentation.

## Difficultés rencontrées

Les principales difficultés ont concerné :

- la mise en place de MongoDB ;
- la configuration de WampServer ;
- la gestion des rôles ;
- le développement des filtres dynamiques ;
- la réalisation des maquettes Figma.

Chaque difficulté a été résolue progressivement grâce à une démarche de recherche, de tests, de corrections et de validation de chaque fonctionnalité avant de passer à la suivante.

## Bilan

Le projet a permis de mettre en œuvre les compétences du titre DWWM :

- développement front-end ;
- développement back-end ;
- gestion de bases de données SQL et NoSQL ;
- sécurisation de l'application ;
- gestion de projet ;
- documentation technique.

Ce projet m'a permis de consolider mes compétences en développement web, en gestion de bases de données, en sécurisation d'une application et en conduite d'un projet complet, de l'analyse jusqu'à la documentation.
